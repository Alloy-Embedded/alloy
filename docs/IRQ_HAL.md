# IRQ HAL

Zero-overhead interrupt abstraction.  All dispatch is routed through a RAM
vector table so handlers can be registered and replaced at runtime without
re-linking.

---

## Enable / Disable

```cpp
#include "hal/irq/irq_handle.hpp"

using P = alloy::device::runtime::PeripheralId;

// Enable USART2 IRQ at priority 4.
alloy::hal::irq::enable<P::Usart2>(4);

// Raise priority while inside a critical section.
alloy::hal::irq::set_priority<P::Usart2>(2);

// Disable.
alloy::hal::irq::disable<P::Usart2>();
```

`enable` / `disable` / `set_priority` are `constexpr`-safe on the call site
(the NVIC write itself is runtime).  They resolve the IRQ number at compile
time via `IrqVectorTraits<P>::kIrqId`.

Calling any of these on a peripheral where `IrqVectorTraits<P>::kPresent ==
false` triggers a `static_assert` at the call site.

## Handler Registration

```cpp
#include "hal/irq/irq_table.hpp"

// Install a lambda / function pointer.
alloy::hal::irq::set_handler<P::Usart2>([]() noexcept {
    // ISR body — runs at interrupt priority; no RTOS calls.
    process_uart_rx();
});

// Remove handler (reverts to do-nothing stub).
alloy::hal::irq::clear_handler<P::Usart2>();
```

Handlers are stored in a 128-entry RAM table (`IrqTable`).  Each entry is
one `IrqHandler` function pointer (4 bytes on 32-bit targets).  Table is
`.data`-initialized to the no-op stub.

**Thread safety**: `set_handler` is not ISR-safe.  Disable the IRQ before
swapping handlers if the ISR may fire during registration.

## ALLOY_IRQ_HANDLER Macro

For ISRs that need a real linker symbol (e.g. DMA streams, fault handlers):

```cpp
ALLOY_IRQ_HANDLER(P::Usart2) {
    // Runs as extern "C" USART2_IRQHandler().
    // Also updates the RAM table entry — both paths work.
    process_uart_rx();
}
```

The macro expands to:

```cpp
extern "C" void USART2_IRQHandler() noexcept { ... }
```

where the vector name is taken from `IrqVectorTraits<P>::kVectorName`.

Use this macro when:
- you want a strongly-linked ISR (no RAM table lookup overhead)
- you need the startup `.s` alias to resolve at link time

Use `set_handler` when:
- the handler is installed after `main()` (RTOS task, dynamic driver load)
- you swap handlers at runtime

## IrqVectorTraits

`IrqVectorTraits<P>` is generated by `alloy-cpp-emit` from the device IR
`irq_table` block.  Include the generated header:

```cpp
#include "<device>/generated/runtime/devices/<device>/irq_vector_traits.hpp"
```

Traits:

| Field          | Type              | Value                             |
|----------------|-------------------|-----------------------------------|
| `kPresent`     | `bool`            | `true` when vector exists         |
| `kIrqId`       | `IrqId`           | `make_irq_id(n)` — IRQ number     |
| `kVectorName`  | `const char*`     | e.g. `"USART2"`                   |

## Default Dispatch Flow

1. Peripheral fires → CPU jumps to `ALLOY_IRQ_SLOT_n_Handler` (weak symbol
   in `default_handler.cpp`).
2. Stub reads `ICSR.VECTACTIVE` → recovers IRQn.
3. Calls `alloy_default_irq_dispatch(irqn)`.
4. `dispatch_irq(irqn)` looks up `IrqTable[irqn]` and calls the handler.

If `ALLOY_IRQ_HANDLER` is used the strongly-linked `USART2_IRQHandler`
overrides the weak slot — step 1 jumps directly to user code.

## Porting to New Architectures

See [PORTING_NEW_PLATFORM.md](PORTING_NEW_PLATFORM.md#irq--architecture-implementation-requirements).

Summary:
- Add `src/hal/irq/detail/<arch>/<arch>_impl.hpp` — implement enable/disable/priority.
- Add `#elif defined(__<arch>__)` block in `default_handler.cpp` for ISR stubs.
- Provide the ISR number recovery (ICSR, MCAUSE, INTLEVEL, etc.).
