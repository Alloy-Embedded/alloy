# Tasks: alloy CLI Distribution

All phases are host-only (no hardware required).

## 1. Package skeleton

- [ ] 1.1 Create `alloy-cli/` repo (or `tools/alloy-cli/` monorepo subdir).
      Add `pyproject.toml` with entry point `alloy = "alloy_cli.__main__:main"`.
      Add `setup.cfg` / `MANIFEST.in`. Publish placeholder to TestPyPI.
- [ ] 1.2 Refactor `scripts/alloyctl.py` into importable modules under
      `alloy_cli/core/` (cmake_runner, probe_runner, board_discovery, device_registry).
      Keep `scripts/alloyctl.py` as thin shim calling `alloy_cli` package for
      backward compatibility during transition.
- [ ] 1.3 Add `alloy_cli/commands/` directory; move each `cmd_*` function into its
      own module file (`new.py`, `build.py`, `flash.py`, `device.py`, `boards.py`,
      `doctor.py`, `bundle.py`).
- [ ] 1.4 Wire `__main__.py` with argparse dispatch to all command modules.
- [ ] 1.5 Add unit tests `alloy-cli/tests/test_board_discovery.py`: exercise
      `_discover_boards(root)` with a temporary board.json fixture.

## 2. `alloy new` command

- [ ] 2.1 Create `alloy_cli/commands/new.py`: scaffold project directory tree.
      Templates in `alloy_cli/data/templates/new_project/`.
- [ ] 2.2 Implement `CMakeLists.txt` template with `ALLOY_BOARD` + `find_package`
      placeholders filled from `board.json`.
- [ ] 2.3 Implement `src/main.cpp` template: blink loop using `pin_handle` for
      the first LED in `board.json leds[]`.
- [ ] 2.4 Implement `.vscode/launch.json` template from `board.json debug` fields.
- [ ] 2.5 Add integration test: run `alloy new --board nucleo_g071rb test_proj`;
      verify file tree created and CMakeLists.txt contains `ALLOY_BOARD=nucleo_g071rb`.

## 3. `alloy device` commands

- [ ] 3.1 Implement `alloy_cli/core/device_registry.py`: fetches
      `alloy-rs.dev/registry/devices.json` (or uses local cache).
- [ ] 3.2 Implement `alloy device add <id>`: download package from registry URL
      to `~/.alloy/devices/`; verify SHA256; unpack.
- [ ] 3.3 Implement `alloy device list [--vendor] [--family] [--cached]`:
      print Rich table of available/cached devices.
- [ ] 3.4 Implement `alloy device info <id>`: print IR coverage, arch, SVD source,
      alloy-devices version from `artifact-manifest.json` inside the package.
- [ ] 3.5 Implement `alloy device prefetch --all`: bulk download all devices in
      registry to local cache (for CI pre-warm and air-gapped environments).

## 4. `alloy boards` command

- [ ] 4.1 Implement `alloy_cli/commands/boards.py`: call `_discover_boards(root)`

      board_id, vendor, family, arch, tier.
- [ ] 4.2 Support `--vendor`, `--family`, `--tier` filters.
- [ ] 4.3 Add `--json` output flag for machine-readable consumption.
