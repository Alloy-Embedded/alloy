# Design: UART Lite — Full Surface

## Problem

`lite::port<P>` has no connector/pin-routing layer: the caller must configure GPIO
alternate-function manually before calling `configure()`.  The old `port_handle<Connector>`
has compile-time pin safety via `connector<USART1, tx<PA9>, rx<PA10>>` but pays a heavy
price: `UartSemanticTraits` (codegen artifact) + `find_runtime_field_ref_by_register_and_offset`
binary search for every register access + virtual-dispatch-style field-id lookups.

Goal: keep the connector syntax the user loves, pay zero overhead at runtime.

---

## Architecture

```
User code
──────────────────────────────────────────────────────────────────
  using Rs485 = alloy::hal::uart::port<
      connector<USART1, tx<PA9>, rx<PA10>, de<PA12>>>;

  Rs485 uart{cfg};
  uart.configure();          // GPIO AF + clock + MMIO init, all at once
  uart.enable_de(true);
  uart.enable_hardware_flow_control(false);
  uart.write(buf);
──────────────────────────────────────────────────────────────────
  hal/uart/port.hpp          — new user-facing type (replaces port_handle<C>)
  ├── connect()              — calls apply_route_operations(C::operations())
  ├── configure()            — calls connect() then MMIO init
  ├── [25 new methods]       — direct MMIO, if constexpr dispatch
  └── impl: lite::port<P>   — existing Tier 2 MMIO kernel

Compile-time data sources
  peripheral_traits.h  ──── base address, IRQ lines, RCC enable/reset, KernelClockMux
  UartSemanticTraits   ──── capability booleans (kSupportsDma, kFifoDepth, …)
  ConnectorTraits      ──── pin→signal→peripheral legality table (generated)
  routes.hpp           ──── AF operations list for apply_route_operations()
```

### Three-layer validation (compiler view)

```
connector<USART1, tx<PA9>, rx<PA10>>
         │
         ▼  (instantiation)
ConnectorTraits<PinId::PA9, PeripheralId::USART1, SignalId::signal_tx>::kPresent
         │
         ├── kPresent == true  → connector is valid, operation list compiled in
         └── kPresent == false → static_assert fires:
               "Invalid connector for USART1 tx. Valid pins: PB6.
                Provenance: stm32-open-pin-data; patches=..."
```

**Guard A (from-USART side, already generated):**
"Invalid connector for USART1 tx. Valid pins: PB6."

**Guard B (from-GPIO side, NEW — modm-style):**
"PA9 cannot serve as signal_tx on USART1. PA9 supports: signal_tx→USART1, signal_tx→USART2."
Generated by adding a `SignalSourceTraits<PinId, SignalId>` layer to connectors.hpp.j2.

**Guard C (signal count/identity, NEW):**
```cpp
static_assert(
    has_tx_or_rx_v<Connector>,
    "uart::port<C> requires at least one tx<> or rx<> binding.");
static_assert(
    !has_duplicate_role_v<Connector>,
    "uart::port<C>: duplicate tx<> or rx<> bindings detected.");
```

## Comparison: alloy vs modm

| Aspect | alloy current (`port_handle<C>`) | modm | alloy new (`uart::port<C>`) |
|--------|----------------------------------|------|-----------------------------|
| Pin-routing syntax | `connector<USART1, tx<PA9>>` | `Uart1::connect<GpioA9::Tx>()` | `connector<USART1, tx<PA9>>` (unchanged) |
| From-USART error | "Valid pins: PB6" ✓ | — | "Valid pins: PB6" (unchanged) |
| From-GPIO error | — | "GpioA9::Tx only connects to Usart1!" | "PA9 as tx only serves USART1, USART2" (new) |
| Signal count check | — | `static_assert(sizeof...(Signals)==2)` | `static_assert(has_tx_or_rx)` (new) |
| connect() separate from init() | — | `connect<>()` + `initialize<>()` | `connect()` + `configure()` (new separation) |
| Register access | field-ref binary search (runtime) | direct MMIO | direct MMIO (lite::port) |
| Capability guards | `UartSemanticTraits::kSupportsDma` | `BufferedUart<Hal4>` code gen | `UartSemanticTraits::kSupportsDma` |
| Missing features | DMA, FIFO, LIN, RS-485, half-duplex | complete | all of the above (25 new methods) |

## `uart::port<Connector>` API sketch

```cpp
namespace alloy::hal::uart {

// User-facing type — replaces port_handle<Connector>
template <typename Connector>
class port {
    // Compile-time validation (Guards A + C)
    static_assert(Connector::valid, "connector<> bindings failed validation.");
    static_assert(has_tx_or_rx_v<Connector>,
        "uart::port requires at least one tx<> or rx<> binding.");

    using P       = Connector::peripheral_type;             // peripheral_traits.h
    using Caps    = UartSemanticTraits<Connector::peripheral_id>;
    using Impl    = lite::port<P>;                          // MMIO kernel

  public:
    // --- Phase 1: existing port_handle API (renamed, same semantics) -------
    [[nodiscard]] auto configure()
        -> core::Result<void, core::ErrorCode>;             // connect() + MMIO init
    [[nodiscard]] auto write(std::span<const std::byte>) const
        -> core::Result<std::size_t, core::ErrorCode>;
    [[nodiscard]] auto read(std::span<std::byte>) const
        -> core::Result<std::size_t, core::ErrorCode>;
    [[nodiscard]] auto flush() const
        -> core::Result<void, core::ErrorCode>;
    [[nodiscard]] static constexpr auto irq_number(std::size_t = 0) noexcept
        -> std::uint32_t;
    static void clock_on()  noexcept;
    static void clock_off() noexcept;
    static void irq_enable()  noexcept;
    static void irq_disable() noexcept;

    // --- Phase 2: connect() separate from configure() (modm pattern) ------
    // Applies GPIO AF + clock enable only.  Call before configure() or
    // separately in a board-init routine.
    static auto connect() -> core::Result<void, core::ErrorCode>;

    // --- Phase 3: new methods (feature parity with old backend) ------------
    // RS-485 / DE
    static void enable_de(bool) noexcept
        requires (Caps::kSupportsRS485);
    static void set_de_polarity(bool active_high) noexcept
        requires (Caps::kSupportsRS485);
    static void set_de_assertion_time(std::uint8_t ticks) noexcept
        requires (Caps::kSupportsRS485 && StModernUsart<P>);
    static void set_de_deassertion_time(std::uint8_t ticks) noexcept
        requires (Caps::kSupportsRS485 && StModernUsart<P>);

    // Flow control
    static void enable_hardware_flow_control(bool) noexcept
        requires (Caps::kSupportsFlowCtl);

    // Half-duplex
    static void set_half_duplex(bool) noexcept
        requires (Caps::kSupportsHalfDuplex);

    // LIN
    static void enable_lin(bool) noexcept
        requires (Caps::kSupportsLin);
    static void set_lin_break_length(bool long_break) noexcept
        requires (Caps::kSupportsLin);
    static void send_lin_break() noexcept
        requires (Caps::kSupportsLin);
    [[nodiscard]] static bool lin_break_detected() noexcept
        requires (Caps::kSupportsLin);
    static void clear_lin_break_flag() noexcept
        requires (Caps::kSupportsLin);

    // Errors
    [[nodiscard]] static auto read_and_clear_errors() noexcept -> UartErrors;
    [[nodiscard]] static auto error_flags() noexcept           -> UartErrors;

    // FIFO (SCI3 only, kFifoDepth > 1)
    static void enable_fifo(bool) noexcept
        requires (StModernUsart<P> && Caps::kFifoDepth > 1u);
    static void set_tx_fifo_threshold(FifoTrigger) noexcept
        requires (StModernUsart<P> && Caps::kFifoDepth > 1u);
    static void set_rx_fifo_threshold(FifoTrigger) noexcept
        requires (StModernUsart<P> && Caps::kFifoDepth > 1u);
    [[nodiscard]] static bool tx_fifo_full() noexcept
        requires (StModernUsart<P> && Caps::kFifoDepth > 1u);
    [[nodiscard]] static bool rx_fifo_full() noexcept
        requires (StModernUsart<P> && Caps::kFifoDepth > 1u);

    // DMA
    static void enable_dma_tx(bool) noexcept
        requires (Caps::kSupportsDma);
    static void enable_dma_rx(bool) noexcept
        requires (Caps::kSupportsDma);

    // IRQ dispatch
    static void enable_interrupt(InterruptKind, bool) noexcept;

    // Smartcard / IrDA
    static void set_smartcard_mode(bool) noexcept
        requires (Caps::kSupportsSmartcard);
    static void set_irda_mode(bool) noexcept
        requires (Caps::kSupportsIrda);

    // Wakeup
    static void enable_wakeup_from_stop(bool) noexcept
        requires (StModernUsart<P> && Caps::kSupportsWakeFromStop);

    // Baudrate
    [[nodiscard]] static auto set_baud_rate(std::uint32_t bps,
                                             std::uint32_t clk_hz) noexcept
        -> core::Result<void, core::ErrorCode>;   // returns Err if bps > kMaxBaudHz

  private:
    Config config_{};
};

}  // namespace alloy::hal::uart
```

## `connect()` implementation

```cpp
template <typename Connector>
auto port<Connector>::connect() -> core::Result<void, core::ErrorCode> {
    return hal::detail::apply_route_operations(Connector::operations());
    // apply_route_operations() iterates over consteval operation list:
    //   - RCC enable  (set APBxENR bit)
    //   - GPIO pinmux (set AFRx + MODER)
    // All operations encoded as consteval RouteOperation entries in routes.hpp.
}
```

## Codegen changes needed

### 1. alloy-devices YAML patches (stm32g0-bootstrap.yaml + stm32f4-bootstrap.yaml)

Add per-USART:
```yaml
  - op: add_semantic
    peripheral: USART1
    key: kSupportsFlowCtl
    value: true       # false for LPUART, USART3/4 on some packages (no CTS/RTS pins)

  - op: add_semantic
    peripheral: USART1
    key: kSupportsRS485
    value: true       # false for LPUART

  - op: add_semantic
    peripheral: USART1
    key: kSupportsIrda
    value: true       # false for LPUART

  - op: add_semantic
    peripheral: USART1
    key: kDmaTxSignal
    value: 53         # DMAMUX channel ID; device-specific

  - op: add_semantic
    peripheral: USART1
    key: kDmaRxSignal
    value: 52
```

### 2. uart.hpp.j2 — add new fields to UartSemanticTraits

Base template fallbacks:
```cpp
static constexpr bool         kSupportsFlowCtl  = false;
static constexpr bool         kSupportsRS485    = false;
static constexpr bool         kSupportsIrda     = false;
static constexpr std::uint8_t kDmaTxSignal      = 0u;
static constexpr std::uint8_t kDmaRxSignal      = 0u;
```

Per-peripheral specialization:
```jinja2
static constexpr bool         kSupportsFlowCtl  = {{ 'true' if periph.semantics.get('kSupportsFlowCtl') else 'false' }};
static constexpr bool         kSupportsRS485    = {{ 'true' if periph.semantics.get('kSupportsRS485') else 'false' }};
static constexpr bool         kSupportsIrda     = {{ 'true' if periph.semantics.get('kSupportsIrda') else 'false' }};
static constexpr std::uint8_t kDmaTxSignal      = {{ periph.semantics.get('kDmaTxSignal', 0) }}u;
static constexpr std::uint8_t kDmaRxSignal      = {{ periph.semantics.get('kDmaRxSignal', 0) }}u;
```

### 3. connectors.hpp.j2 — add from-GPIO SignalSourceTraits (modm Guard B)

```jinja2
// For each (pin, signal_id) where at least one legal peripheral exists:
template<>
struct SignalSourceTraits<PinId::PA9, SignalId::signal_tx> {
    // List of valid peripherals for this pin+signal combination:
    static constexpr std::array kValidPeripherals = {
        PeripheralId::USART1,
    };
    static constexpr auto kMessage =
        "PA9::signal_tx can only connect to: USART1. "
        "Check the connector<> peripheral ID or choose a different pin.";
};

// In the primary (catch-all) specialization that fires on mismatch:
template <PinId Pin, PeripheralId Peri, SignalId Sig>
struct ConnectorTraits {
    static constexpr bool kPresent = false;
    // Guard A (from-USART side) already exists.
    // Guard B (from-GPIO side): fires when pin is valid but peripheral is wrong.
    static_assert(
        !SignalSourceTraits<Pin, Sig>::kPresent ||
         is_valid_in<Peri>(SignalSourceTraits<Pin, Sig>::kValidPeripherals),
        SignalSourceTraits<Pin, Sig>::kMessage);
};
```

## Migration path: port_handle<C> → uart::port<C>

| Old | New |
|-----|-----|
| `hal::uart::port_handle<Rs485Connector>` | `hal::uart::port<Rs485Connector>` |
| `port_handle::configure()` | `port::configure()` (now also calls connect()) |
| `port_handle::irq_numbers()` → SemanticTraits | `port::irq_number(0)` → `P::kIrqLines[0]` |
| `has_signal<PH, role_id::tx>()` in backend | `uart::port<C>` enforces via requires clause |
| `apply_route_operations()` inside backend | `port::connect()` public static method |
| `UartSemanticTraits::kSupportsDma` | `UartSemanticTraits<C::peripheral_id>::kSupportsDma` |
| `find_runtime_field_ref_by_register_and_offset` | deleted — direct MMIO in `lite::port<P>` |
